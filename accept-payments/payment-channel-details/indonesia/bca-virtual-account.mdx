![](https://www.xendit.co/wp-content/uploads/2021/07/logo-bca.png)BCA Virtual Account is a payment channel provided by Bank Central Asia (BCA), enabling customers to pay conveniently via unique virtual account numbers assigned to each transaction. With its extensive customer base, BCA is a leading financial institution in Indonesia, making this payment method widely recognized and trusted.

When checking out with BCA Virtual Account, customers receive a virtual account number specific to their transaction. They can complete the payment via mobile banking, internet banking, or ATM. Once the payment is made, you receive an instant confirmation, ensuring a seamless and secure payment experience for both you and your customers.

## Payment flow

Mobile banking

Log In To Your Account

1. Open BCA Mobile App
2. Select "m-BCA", then select "m-Transfer"

Payment Details

1. Select "m-BCA", then select "m-Transfer"
2. Enter your Virtual Account Number, for example 3816523906568, then press "OK"
3. Click on "Send" button at the top right corner to proceed
4. Click "OK" to proceed
5. Enter your PIN to authorize the transaction

Internet banking

Log In To Your Account

1. Login to KlikBCA Individual (&lt;https://ibank.klikbca.com&gt;)
2. Select "Transfer", then select "Transfer to BCA Virtual Account"

Payment Details

1. Enter the Virtual Account Number, for example 3816523906568
2. Select "Continue" to proceed your payment
3. Enter "RESPON KEYBCA APPLI 1" shown in your BCA Token, then click on the "Send" button
4. Enter the authentication token code

ATM

Find Nearest ATM

1. Insert your BCA ATM card and PIN
2. Enter your ATM PIN

Payment Details

1. Select Menu "Other Transaction"
2. Select "Transfer"
3. Select "To BCA Virtual Account"
4. Enter Virtual Account Number, for example 3816523906568. Press "Correct" to proceed
5. Verify Virtual Account details and then enter amount to be transferred and select "Correct" to confirm
6. Confirm your transaction details displayed
7. Select "Yes" if the details are correct or "No" if the details are not correct

## Deeplink feature

This feature enables a streamlined payment experience for BCA Virtual Account users by providing a direct deeplink to the My BCA mobile app. When customers receive a deeplink, they can complete their payment with fewer steps, improving conversion rates and user satisfaction.

### Who Can Use This Feature?

BCA VA Deeplink is currently available only to merchants who meet both of these requirements:

1. You are already using BCA Virtual Account as a payment method

2. Your BCA Virtual Account is set up with switcher fund flow

If you meet both requirements and want to activate BCA VA Deeplink, please contact us at [help@xendit.co](mailto:help@xendit.co).

> Pilot Feature Notice
>
> This is a pilot feature. You may experience changes to the API or functionality as we continue development. We recommend testing thoroughly and monitoring our documentation for updates.

### What is BCA VA Deeplink?

BCA VA Deeplink is an enhancement to the standard BCA Virtual Account payment flow that provides customers with a direct link to complete their payment in the My BCA mobile app. Instead of manually copying the virtual account number and navigating through the app, customers can simply tap the deeplink to be taken directly to a pre-filled payment screen.

Key benefits are a reduced payment friction with fewer manual steps a faster checkout experience for mobile users.

### How It Works

When you create a BCA Virtual Account payment request, the API response includes both the standard virtual account number and a deeplink URL in the `actions` array. Here's the user journey:

1. Customer initiates checkout: Customer selects BCA VA as payment method

2. Deeplink generation: You receive both VA number and deeplink URL in the API response

3. Customer interaction: Present the deeplink to the customer (as a button, QR code, or clickable link)

4. Deeplink activation: Customer taps the deeplink on their mobile device

5. Redirect flow:

- Opens mobile browser first

- Automatically redirects to My BCA app (or app store if not installed)

6. Pre-filled payment: My BCA app opens with payment details pre-populated

7. Payment completion: Customer reviews and confirms payment

8. Return to merchant: Customer is redirected to your `success_return_url`

### Integration Guide

1. Creating a Payment Request with Deeplink

Include the `success_return_url` in your `channel_properties` to ensure customers are redirected back to your application after completing payment.

2. Understanding the Response

The response includes two action objects:

- `VIRTUAL_ACCOUNT_NUMBER`: The standard VA number for fallback payment methods (internet banking, ATM)

- `DEEPLINK_URL`: The deeplink that redirects customers to My BCA app

### Important Considerations

Deeplink Expiration

The deeplink has a 5-minute expiration from the time it's generated. This is separate from the virtual account expiration, which typically lasts longer.

- Deeplink expiry: 5 minutes

- Virtual account expiry: Defined by `expires_at` in `channel_properties`

After the deeplink expires, customers can still complete payment using the virtual account number through standard methods (mobile banking, internet banking, ATM).

App Compatibility

The deeplink only works with the My BCA mobile app:

- ✅ My BCA (new app)

- ❌ BCA Mobile (legacy app)

Platform Support:

- iOS (App Store)

- Android (Google Play Store)

Best Practice: Always display both the deeplink AND the virtual account number to ensure customers have a fallback option.

### Testing Your Integration

Sandbox Environment

In sandbox, you can test creating payment requests and simulating payments, but you cannot test the actual deeplink functionality (the deeplink will not open the My BCA app in test mode).

What you can test in sandbox:

1. Create a payment request using the sandbox API endpoint

2. Verify the response contains both actions (VA number and deeplink URL)

3. Simulate a successful payment using the payment simulation endpoint

Simulating a Payment:

Request - POST /v3/payment\_requests/&#123;payment\_request\_id&#125;/simulate

```json
{
    "amount": 12000
}
```

Response

```json
{
    "status": "PENDING",
    "message": "A simulated payment for the specified payment request has been queued"
}
```

After simulating the payment, you will receive a webhook notification similar to what you would receive in production.

What You Cannot Test in Sandbox

- The actual deeplink redirect flow (browser to My BCA app)
- My BCA app opening with pre-filled payment details
- App store redirect for users without My BCA installed
- The `success_return_url` redirect after payment

These features can only be fully tested in production with real transactions.
