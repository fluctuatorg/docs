To facilitate testing of card transactions, we provide a range of test card numbers. These numbers vary based on the country of operation and your live connection. For specific guidance on which test card numbers to use for your integration, please contact Xendit Customer Support.

## Test card numbers

For any card, use any CVV and make sure the expiry date is in the future.

## Simulate a 3DS challenge authentication flow result

To test different outcomes for 3DS challenge flows:

1. Create a payment request with `should_authenticate = true`.
2. Navigate to the URL provided by `payer_authentication_url`.
3. On the MPGS ACS Simulator page, select one of the following options:

| Authentication option | Status |
| --- | --- |
| AUTHENTICATED | SUCCESS |
| UNAUTHENTICATED | FAILED |
| CANCELLED\_AUTHENTICATION | FAILED |
| AUTHENTICATION\_NOT\_AVAILABLE | FAILED |
| AUTHENTICATION\_REJECTED | FAILED |
| AUTHENTICATION\_SERVER\_ERROR | FAILED |
| API\_GW\_ASM\_POLICY\_ERROR | FAILED |
